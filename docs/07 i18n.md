# Internationalisation (i18n)

## Vue d'ensemble

```
src/lib/i18n.js               ← Configuration i18next
    │
    ├─ i18next-http-backend   ← Chargement paresseux des fichiers JSON
    ├─ i18next-browser-languagedetector ← Détection automatique de la langue
    └─ react-i18next          ← Hooks et composants pour React

public/locales/
    ├─ fr/   ← Français (langue de référence)
    ├─ en/   ← Anglais
    ├─ ar/   ← Arabe (RTL)
    └─ he/   ← Hébreu (RTL)
```

La langue active est persistée dans `localStorage` (clé `i18nextLng`). Le namespace par défaut est `common`. Chaque page ou domaine fonctionnel possède son propre namespace.

---

## Configuration i18next

```js
// src/lib/i18n.js
export const LOCALES = ['en', 'fr', 'ar', 'he']
export const DEFAULT_LOCALE = 'en'

i18n
  .use(HttpBackend)           // chargement HTTP des JSON
  .use(LanguageDetector)      // détecte la langue du navigateur
  .use(initReactI18next)      // intégration React
  .init({
    fallbackLng: DEFAULT_LOCALE,
    supportedLngs: LOCALES,
    interpolation: { escapeValue: false },  // React gère l'échappement XSS
    debug: import.meta.env.DEV,
    backend: {
      loadPath: '/locales/{{lng}}/{{ns}}.json',
    },
  })
```

## Utilisation dans les composants

### Hook de base

```jsx
import { useTranslation } from "react-i18next"

function LoginScreen() {
  const { t } = useTranslation("auth")

  return <h1>{t("login.title")}</h1>
  // → "Connexion à votre compte" (fr)
  // → "Log in to your account"  (en)
}
```

### Plusieurs namespaces

```jsx
const { t } = useTranslation(["auth", "auth-extra"])

// Namespace explicite avec préfixe
<Link>{t("auth-extra:adminLoginLink")}</Link>

// Namespace implicite (premier de la liste)
<h1>{t("login.title")}</h1>
```

### Interpolation de variables

```jsx
// Dans le JSON : "appHint.description": "Code pour {{email}}"
t("appHint.description", { email: "admin@cyna.fr" })
// → "Code pour admin@cyna.fr"
```

### Pluralisation

```jsx
// Dans le JSON : "items_one": "{{count}} article", "items_other": "{{count}} articles"
t("items", { count: 3 })
// → "3 articles"
```

## Support RTL (Arabe & Hébreu)

La direction du texte est gérée par `DirectionProvider` dans `main.tsx` :

```jsx
// main.tsx
function Root() {
  const { i18n } = useTranslation()
  const dir = i18n.dir(i18n.language)  // "rtl" pour ar/he, "ltr" sinon

  return (
    <DirectionProvider direction={dir}>
      <App />
    </DirectionProvider>
  )
}
```

`DirectionProvider` (shadcn/ui) applique l'attribut `dir` sur le conteneur racine. Les composants Radix UI respectent automatiquement la direction.

Le changement de langue via `LangSwitcher` (`src/components/layout/lang-switcher.jsx`) appelle `i18n.changeLanguage(lang)` et recharge la direction sans rechargement de page.

---

## Changer de langue — LangSwitcher

```jsx
// src/components/layout/lang-switcher.jsx
import i18n, { LOCALES } from "@/lib/i18n"

function LangSwitcher() {
  const handleChange = (lang) => {
    i18n.changeLanguage(lang)  // persiste dans localStorage
  }
  // Affiche un select/combobox avec LOCALES = ['en', 'fr', 'ar', 'he']
}
```

## Ajouter un namespace

### 1. Créer les fichiers JSON dans chaque langue

```json
// public/locales/fr/notifications.json
{
  "title": "Notifications",
  "empty": "Aucune notification",
  "markRead": "Marquer comme lu",
  "markAllRead": "Tout marquer comme lu"
}
```

```json
// public/locales/en/notifications.json
{
  "title": "Notifications",
  "empty": "No notifications",
  "markRead": "Mark as read",
  "markAllRead": "Mark all as read"
}
```

### 2. Utiliser dans le composant

```jsx
const { t } = useTranslation("notifications")
<h2>{t("title")}</h2>
```

> Les fichiers sont chargés **à la demande** (lazy) au premier usage du namespace — pas de configuration supplémentaire dans `i18n.js`.
